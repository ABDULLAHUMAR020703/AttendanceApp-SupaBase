## ‚úÖ Replacement Checklist

### Step 1: Prepare New Logo Assets

Create new logo files with the following specifications:

#### Primary Logo (UI Usage)
- **File:** `apps/mobile/assets/logo.png`
- **Format:** PNG with transparency
- **Recommended Size:** 1024x1024px (square, will be scaled by Logo component)
- **Aspect Ratio:** Square (1:1)
- **Background:** Transparent

#### App Icon (Standard)
- **File:** `apps/mobile/assets/icon.png`
- **Format:** PNG
- **Recommended Size:** 1024x1024px
- **Aspect Ratio:** Square (1:1)
- **Background:** Solid color or transparent
- **Note:** Used for Android, iOS, and notifications

#### Android Adaptive Icon
- **File:** `apps/mobile/assets/adaptive-icon.png`
- **Format:** PNG
- **Recommended Size:** 1024x1024px
- **Aspect Ratio:** Square (1:1)
- **Safe Zone:** Keep important content within 512x512px center (will be masked)
- **Background:** Transparent (background color set in `app.json`)

#### Splash Screen
- **File:** `apps/mobile/assets/splash.png`
- **Format:** PNG
- **Recommended Size:** 1242x2436px (iPhone X) or 2048x2732px (iPad)
- **Aspect Ratio:** Portrait (9:16 or 3:4)
- **Background:** Solid color (matches `splash.backgroundColor` in `app.json`)

#### Web Favicon
- **File:** `apps/mobile/assets/favicon.png`
- **Format:** PNG or ICO
- **Recommended Size:** 32x32px or 64x64px
- **Aspect Ratio:** Square (1:1)

### Step 2: Replace Asset Files

Replace the following files in `apps/mobile/assets/`:

- [ ] `logo.png` - UI logo (used in screens and navigation)
- [ ] `icon.png` - App icon (Android, iOS, notifications)
- [ ] `adaptive-icon.png` - Android adaptive icon foreground
- [ ] `splash.png` - Splash screen image
- [ ] `favicon.png` - Web favicon
- [ ] (Optional) `logo.svg` - SVG version if needed

### Step 3: Update Configuration (if needed)

#### Update Splash Background Color (if changed)
- **File:** `apps/mobile/app.json`
- **Line:** 12
- **Action:** Update `splash.backgroundColor` if your new splash screen uses a different background

#### Update Adaptive Icon Background Color (if changed)
- **File:** `apps/mobile/app.json`
- **Line:** 27
- **Action:** Update `android.adaptiveIcon.backgroundColor` if your new adaptive icon needs a different background

### Step 4: Clean Build Artifacts

#### For Android:
```bash
cd apps/mobile
# Clean Android build
rm -rf android/app/build
rm -rf android/.gradle
# Or on Windows:
# rmdir /s /q android\app\build
# rmdir /s /q android\.gradle
```

#### For iOS:
```bash
cd apps/mobile
# Clean iOS build
rm -rf ios/build
# Or on Windows (if using WSL):
# rm -rf ios/build
```

#### Clear Expo Cache:
```bash
cd apps/mobile
npx expo start --clear
```

### Step 5: Regenerate Native Icons (EAS Build)

If using EAS Build, icons are automatically regenerated during build. For local builds:

#### Android:
```bash
cd apps/mobile
npx expo prebuild --clean
```

This will regenerate:
- Android mipmap icons (all densities)
- Android adaptive icons
- Android notification icons
- Android splash screen logos

#### iOS:
```bash
cd apps/mobile
npx expo prebuild --clean
```

This will regenerate:
- iOS AppIcon.appiconset (all sizes)

### Step 6: Verify Changes

#### Test UI Logos:
1. Open the app
2. Check Login screen - logo should appear in header
3. Check Drawer menu - logo should appear in drawer header
4. Check all screens that use Logo component

#### Test App Icons:
1. **Android:** Build APK and install - check app launcher icon
2. **iOS:** Build and install - check home screen icon
3. **Notifications:** Send a test notification - check notification icon

#### Test Splash Screen:
1. Close app completely
2. Reopen app - splash screen should display new logo
3. Check both Android and iOS

### Step 7: Rebuild and Deploy

#### Development:
```bash
cd apps/mobile
npm start
# Press 'a' for Android or 'i' for iOS
```

#### Production Build (EAS):
```bash
cd apps/mobile
eas build --platform android
eas build --platform ios
```
